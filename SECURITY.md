# Security Policy

## 支持版本

当前仅保证 `main` 分支最新提交的安全修复。

## 漏洞上报

如果你发现安全问题（密钥泄露、鉴权绕过、远程执行风险等），请不要在公开 Issue 直接披露细节。

请通过以下方式私下联系维护者：

- GitHub Security Advisory
- 或通过仓库维护者联系方式（后续补充）

## 最佳实践

1. 所有密钥使用环境变量注入，不写入代码仓库。
2. 生产环境不要使用 `gateway.auth.mode=none`。
3. Mem0 API 建议仅开放内网，并在网关层增加鉴权。
4. 定期轮换 Kimi / 飞书等第三方密钥。
